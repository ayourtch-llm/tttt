The Problem: AI Agents Hit Walls

Lessons from an 8+ hour orchestration session (Opus coordinating Sonnet via tmux)

  • 40% of tokens wasted on polling loops → "sleep 5 && tail" is all you have

  • No structured output detection → Can't wait for specific patterns
  • Rate limits kill unattended sessions → No automatic recovery

Key Design Choices

  • PtyBackend trait for mock/real/restored PTYs
  • Shared SessionManager across TUI + MCP
  • Non-blocking reads with poll()
  • Render debouncing (50ms idle / 200ms max)

Live Reload

  • SIGUSR1: execv() in-place, preserves all FDs
  • SIGUSR2: Full restart with --resume
  • Same PID, children don't notice
  • Enables development while running

Operational Highlights

Rate Limit Auto-Recovery

tttt_pty_handle_rate_limit detects Claude Code rate limit dialogs, parses the reset timestamp, sleeps until ready, and auto-continues. Unattended sessions survive overnight.

Smart Waiting

tttt_pty_wait_for blocks until a regex matches on screen. tttt_pty_wait_for_idle waits for output to stop. No more "sleep && poll" loops burning tokens.

Remote Attach

tttt attach connects from another terminal via Unix socket. Watch AI agents work in real-time with full color rendering. Multiple viewers supported simultaneously.

Pattern-Triggered Actions

tttt_notify_on_pattern watches any session for regex matches and auto-injects text responses. Recurring watchers handle permission prompts, confirmations, and more.
